# Migrating from 0.4 to 0.5

## mruby

Updated to 4.0.0. You can see the breaking changes in [the release
notes](https://mruby.org/releases/2026/04/20/mruby-4.0.0-released.html).

## Raylib

Updated to 6.0. You can see the breaking changes in [the release
notes](https://github.com/raysan5/raylib/releases/tag/6.0).

## Emscripten

Updated to 6.0.0. You can see the breaking changes in [the release
notes](https://github.com/emscripten-core/emscripten/releases/tag/6.0.0).

## CLI

Renamed `--input` to `--entrypoint`.

## `taylor-config.json`

The `input` key has been renamed to `entrypoint`. `input` will continue to be
respected but you should update. New projects are generated with `entrypoint`
instead.

I have deprecated the following keys:

- `export-directory`
- `export-targets`
- `load-paths`
- `copy-paths`

In favour of:

- `export_directory`
- `export_targets`
- `load_paths`
- `copy_paths`

## Texture2D#draw

The origin no longer defaults to the centre of the destination.
